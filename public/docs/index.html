What is SMSaaS?

SMSaaS is a multi-tenant bulk SMS platform built specifically for Maldivian businesses. The platform operator owns a single Twilio account, purchases SMS at wholesale rates, and resells to business clients through a clean web interface.

Each client gets their own branded console â€” a web application they can log into, upload their customer contacts, organise them into groups, and send bulk SMS campaigns. They never interact with Twilio or any technical infrastructure. From their perspective, it just works.

Campaign send flow

1
Business clicks Send in their Console
The Console sends the campaign details to the SMS API Server â€” group ID, message body, and the tenant's API key for authentication.
2
Server validates and deducts credits
The server checks the Tenant has enough credits. If yes, credits are deducted immediately before any messages are sent â€” preventing overspend.
3
Campaign and messages created in database
A Campaign record is created with status RUNNING. One Message record is created per Contact in the Group, all starting as QUEUED.
4
Jobs pushed to BullMQ queue
Every message is added to a Redis-backed job queue. The server immediately responds to the Console: "Campaign launched, X messages queued."
5
Worker sends via Twilio
A background worker processes jobs from the queue â€” 10 concurrently, up to 100/second. Each message is sent through the Twilio Messaging Service.
6
Twilio routes to Dhiraagu or Ooredoo
Twilio determines the carrier for each number and routes accordingly. The message arrives on the customer's handset.
7
Delivery status webhook
Twilio POSTs a status callback to the server when each message is delivered or fails. The server updates the Message record in the database.
8
Console shows live results
The Console polls the progress endpoint every 2â€“3 seconds and displays a live progress bar with sent, delivered, failed, and pending counts.

Each Tenant gets their own Console â€” a separate Next.js application deployed independently to Vercel. One API key connects it to the central server. This means:

  • Each console can be branded with the client's business name and logo
  • Client gets their own URL â€” e.g. sms.reethicafe.mv
  • Adding a new client = deploy a new Vercel project + set one environment variable
  • All clients share the same backend server â€” infrastructure cost stays flat

Credits & Billing

How credits work

  • 1 credit = 1 SMS sent to 1 person (for messages up to 160 characters)
  • Credits are pre-purchased â€” deducted before the campaign sends
  • Failed messages (after 3 retries) are automatically refunded
  • Cancelled campaigns refund all unprocessed credits immediately
  • Credits never expire
